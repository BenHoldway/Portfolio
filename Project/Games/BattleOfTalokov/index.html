Battle Of Talokov is an angled top down tank based game, where the player must fight through groups of enemies to complete an objective.

The game is based in a scifi apocalyptic game, where the company of security for humanity is at war with a secret organisation, of which little is known about them.

The player is tasked with infiltrating one of the organisation's bases, fight through any enemy unit they come across, and find the main power core. Once destroyed, the player will return to where they started and be extracted, finishing the level, and the game.

Battle Of Talokov is a short level of a game, developed in UE5.1, in my first university year. The game started from a template project which had all of the assets I needed. The purpose of this project was learning about Game Mechanics, and efficient ways of implementing them.

Project Development

The module had me develop a level of a 3D top down tank game. I decided to build a level which had a story behind it; with the player infiltrating an enemy base to destroy the core power of the facility. Along the level, the player had a range of unique weapons at their disposal to find and use, to fight a bunch of different enemies.

Most of the project's development was spent learning how to use UE and the usage of Blueprints to create functionality. We had the basics of the project setup and player movement in the template that we were given, and was led through improving that and implementing more classes in. The module went through player functionality like shooting and projectiles, as well as adding more enemy types and then VFX and lighting effects to add in later on.

The main process of development was adding the extra weapon functionality and ability to hold 2 different weapon types at once, switching the held one out when a new type was picked up. As well as this, developing the level and setting up the progressing for the player to advance through until completion.

The biggest obstacle that I was able to tackle was the boss fight that I had designed. It had 3 different phases, all with different attacks and a barrier, which the player has to deactivate, between each phase. These phases were handled with a tracker, which allowed for different attacks to be selected. During this fight, a range of enemies continuously spawn for the player to fight.

While I had some experience with using Unreal Engine beforehand, this was the first real project I worked on in the engine. I was able to effectively build base classes for different enemies and weapons, and create child classes of these to build unique utilities and objects throughout the game. With these bases, the game could be further developed and expanded on later.
