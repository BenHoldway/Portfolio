Hidden Secrets is a horror survival game, which the player must survive from an AI robot stalking them down. If they are caught and killed, a life will be taken away from a total of 3 lives.

Be aware of the building's power and oxygen going offline, otherwise you may have problems that you have to deal with. Venture out from the starting room to find the generators to turn them on, but be careful of anything that may be looking for you.

This is the first project which I used C++ with UE5 to develop a game, so a lot of the process was learning the best ways to link code up to the engine. From learning about how to set up player and inputs, as well as components, UI and communication between classes and actors, we were taught the basics before working on our own design.

Game Features

General

  • Implementation of systems using C++
  • Linkage of C++ to Unreal Systems

Player Systems

  • Simple Movement like Walking, Sprinting and Crouching
  • Interactions with objects to progress the game
  • A toggleable flashlight
  • A 3 life system

AI Systems

  • Usage of Behaviour Trees, Custom Tasks, and EQS Systems
  • Random movement around the map
  • Detection of the Player using Unreal's Sight Sense
  • Ability to attack and take a life from the player

In-Game System Tracker

  • Power and Oxygen value trackers
  • Ability for these systems to "fail"
  • Interactables to reset system generation when failing
  • In-Game visible UI to display these values

Project Development

Beginning the module, I had no knowledge of how to use C++ with UE5. Having some basic knowledge of the C++ base language helped the learning process. We began by setting up the GameMode and Player actors, creating basic Player Movement and Input detections.

Outside of the player, we worked on other systems to set up a simple project starting point. These systems included a Game Manager systems to handle targets for the player to shoot, as well as an AI system to allow for an entity to move around using UE5's EQS and Behaviour Tree.

Taking this basic project that we had built up, I was able to direct the game towards what I had designed. A stealth and timed based survival game where the player had to maintain in game systems to keep progressing. I was able to expand the Player and AI systems to add functionality, like stamina for when the player sprints, interactions with their functionality, and a detection system for the AI to make the entity move towards the player when in sight.

On top of these expansions to already created systems, I also worked through the process of creating the in-game power and oxygen trackers. I was able to develop this so it was linked into the core mechanics of the game, as well as to the interactables and in-game UI visuals. Further additions to this was a toggleable flashlight, C++ implemented audio and dust particle VFX effects.

Now having experience of working with C++ in game development, I got an insight to how using it would be within the industry. While I have used code a lot to develop in Unity, using another language which I don't have much knowledge on gave me a fresh perspective on the development process in general.

I took this project further than my other Unity projects, including being in a 3D environment, as well as fully implemented AI which interacts with the environment and player, as well as audio and VFX effects which is not something I know much of.
