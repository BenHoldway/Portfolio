You work a boring job as a cleaner for an office space, cleaning up messes and closing up the space. Find and complete the needed tasks during your shift, from cleaning up to turning electricals off.

Each task will reward you with money at the end of your shift, but make sure there are no remaining tasks at the end of your time. Earn more money every day to support your family, otherwise their health may deteriorate.

This project was created for my 1st year 'Game Prototyping' Module, using Unity to create a small game with a prompt of "Simulating the Mundane".

This was the 2nd part of this module, after creating 'Chasm' in the 1st part, with a development time of 6 weeks. The intention of this was to use the Agile Development Methodology to acurately plan and complete sprints with a larger project than the 3 week game previously created for this module.

Game Features

General

  • Usage of Agile Development Weekly Sprints
  • Full Project Planning and Maintaining
  • Usage of documentation, planning and Gantt Charts

Player

  • Simple Movement
  • Press and Hold Interactions with Objects
  • Slight camera movement around the player with mouse movement

Shift System

  • Shift tracker to handle difficulty of shift tasks
  • A timer that ticks up to the end of the shift day
  • Task generation for the player to complete during the shift

Family and Money

  • Stats for each family member
  • Money value which increases from shift and decreases from purchases
  • Toggleable costs to let the player select what they want to pay
  • Costs which automatically enable/disable if they're needed

Project Development

The development of this project allowed me to take what I had learnt in the 3 week project and apply it to a larger scaled version. Initially, I had a lot of issues coming up with ideas which matched the prompt we were given. They wanted a dull job, but "game-ified", which I really struggled to grasp. After a lot of time, I was able to design the idea which I went with: a 2D top-down cleaning job and money management simulator.

To properly follow a sprint plan for Agile Development, I went through the entire process of planning the project as a whole. I made a list of all of the mechanics within the game, and the tasks I needed to complete. From creating a Gantt chart for the project timeline, as well as using a planning tool to manage which stage tasks are in - To Do, In Progress and Completed.

Through the development process, I made very good progress with the content I was working on. While there were some task movement between different sprints, I was able to keep up to date and manage each week's sprint accurately to manage the development progression as a whole. However, there was too much content in the original plan.

Originally I had a lot more mechanics planned, like events - such as apartment change offers, or different extra costs that would need to be paid. However, due to the short time development, these were cut through the process to ensure that the final project was completed for submission.

This module taught me a lot, from effective time management, as well as good ways to structure and track the development of the project. All these skills made the creation of this game a lot smoother and allowed for me to react when development got slightly behind.

I did make some mistakes in this project while trying to improve my code efficiency, however, which I have since learnt from and expanded my knowledge on more efficient Unity C# paradigms. This project specifically helped me grow a lot as a programmer and really showed me what my strengths and weaknesses are.
